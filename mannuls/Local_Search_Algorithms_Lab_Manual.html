Introduction — Why Local Search?

The Problem with Being a Perfectionist

Imagine you're at a massive food festival with 10,000 stalls. You want the tastiest biryani. One approach: taste biryani at every single stall (exhaustive search). By stall #347 you'd be in the hospital. The smarter approach? Start somewhere, taste the biryani, look at nearby stalls, and keep moving toward the one that smells better. That's local search.

🧠 Core Idea
Local search algorithms start from an initial state and iteratively move to neighbor states. They don't keep track of paths or an explored set. They use almost constant memory and can find good (often optimal) solutions in huge or infinite state spaces where systematic algorithms would fail.

When Do We Use Local Search?

Local search shines when: (1) the goal state itself is the solution (e.g., find the best arrangement), (2) the search space is astronomically large (TSP, scheduling, VLSI design), (3) you need a good-enough answer fast, and (4) you have limited memory.

Key Vocabulary

TermMeaningReal-World Analogy
StateA complete configurationYour current position on a mountain
NeighborA state reachable by a small changeOne step in any direction
Objective FunctionEvaluates how good a state isThe altitude at your position
Global OptimumThe absolute best stateThe tallest peak on the entire range
Local OptimumBest among neighbors, but not globallyTop of a small hill — great until you see Everest
PlateauFlat region, no neighbor is betterA tableland — no gradient to follow
RidgeNarrow elevated region, hard to navigateA knife-edge — one wrong step and you descend

The State-Space Landscape

Think of the search space as a 3D landscape. Each point is a state, height is the objective value. Our job: find the highest peak (maximization) or the deepest valley (minimization). Every algorithm in this lab is a different strategy for hiking this landscape.

Hill Climbing — The Greedy Mountaineer

🎭 The Analogy
You're blindfolded on a mountain range. You can only feel the ground under your feet and one step in each direction. You always step uphill. When there's no uphill step? You declare: "I've reached the top!" (Spoiler: you might be on a tiny hill while Everest is 2 km away. 😬)

How It Works (Step by Step)

Step 1: Pick a random starting point
↓
Step 2: Look at all neighboring states
↓
Step 3: Is any neighbor BETTER than where I am now?
YES → move to the best neighbor   |   NO → STOP (you're stuck!)
Return: current state (local optimum — maybe global, maybe not)

Pseudocode

function HILL-CLIMBING(problem):
  current ← problem.INITIAL-STATE
  loop:
    neighbor ← highest-valued successor of current
    if VALUE(neighbor) ≤ VALUE(current):
      return current   // stuck — no neighbor is better
    current ← neighbor

Variants

VariantHow It Picks Next StateProsCons
Steepest-AscentEvaluates ALL neighbors, picks the bestMost thorough per stepSlow when many neighbors
First-ChoiceGenerates random neighbors until one is betterFast with huge neighbor setsMay miss the steepest path
StochasticRandomly picks among uphill neighborsExplores more diverse directionsCan be slower to converge
Random-RestartRuns HC multiple times from random startsIf p = success prob, expected restarts = 1/pRequires many runs
⚠️ The Three Villains of Hill Climbing
1. Local Maxima — A peak that isn't the highest. All neighbors are worse, so HC stops.
2. Plateaus — Flat regions. No neighbor is better. HC has no gradient to follow.
3. Ridges — Narrow elevated paths. Neighbors slope down on both sides. Progress requires moving sideways first.

# =====================================================
#  HILL CLIMBING — The simplest local search algorithm
# =====================================================
#  WHAT IT DOES:
#    Start somewhere random → look around → move to the
#    best nearby spot → repeat until stuck.
#
#  WHEN TO USE:  Quick answers, smooth landscapes.
#  WEAKNESS:     Gets stuck at local optima (small hills).
# =====================================================

import random
import math


# ----- STEP 1: Define the problem -----
# This is the "landscape" we're searching.
# It's a wavy function with MULTIPLE PEAKS — some tall, some short.
# Our goal: find the x value that gives the HIGHEST output.

def objective(x):
    """
    The function we want to MAXIMIZE.
    
    Think of it as a mountain range:
       - Input x  = your position (where you're standing)
       - Output   = the altitude at that position
       - GOAL     = find the position with the HIGHEST altitude
    
    This particular function has ~5 peaks between x=0 and x=20,
    which makes it perfect for showing how HC gets stuck.
    """
    return math.sin(x) * math.cos(x / 2) * math.sin(x / 3)


# ----- STEP 2: Define what a "neighbor" means -----
# A neighbor is a point CLOSE to where we are now.
# We generate it by adding a small random nudge to our position.

def get_neighbor(x, step_size=0.1):
    """
    Generate a neighbor of x.
    
    Example: if x = 5.0 and step_size = 0.1
             neighbor could be anywhere from 4.9 to 5.1
    
    WHY random?  So we explore different directions each time.
    WHY small?   Big steps would skip over peaks.
    """
    nudge = random.uniform(-step_size, step_size)
    return x + nudge


# ----- STEP 3: The Hill Climbing algorithm itself -----

def hill_climbing(lower=0, upper=20, max_iterations=1000, step_size=0.1):
    """
    Hill Climbing (Steepest-Ascent style for 1D problems).
    
    ALGORITHM IN PLAIN ENGLISH:
        1. Stand at a random spot
        2. Look around (check a neighbor)
        3. If the neighbor is HIGHER → walk there
        4. If the neighbor is LOWER  → stay put
        5. Repeat until you can't go higher
    
    PARAMETERS:
        lower, upper    = boundaries of our search space [0, 20]
        max_iterations  = safety limit (stop even if not stuck)
        step_size       = how far to look for neighbors
    
    RETURNS:
        best_x     = the x-coordinate of the peak we found
        best_value = the height at that peak
        path       = list of (x, value) showing our journey
    """

    # === INITIALIZATION ===
    # Pick a random starting point (like being dropped blindfolded)
    current_x     = random.uniform(lower, upper)
    current_value = objective(current_x)

    # Keep a record of everywhere we've been (for plotting later)
    path = [(current_x, current_value)]

    # === MAIN LOOP ===
    for step in range(max_iterations):

        # Look at a nearby point
        neighbor_x = get_neighbor(current_x, step_size)

        # Make sure we don't wander outside the search space
        neighbor_x = max(lower, min(upper, neighbor_x))

        # How good is this neighbor?
        neighbor_value = objective(neighbor_x)

        # THE KEY DECISION: Is the neighbor better?
        if neighbor_value > current_value:
            # YES → Move there! This is the "climbing" part.
            current_x     = neighbor_x
            current_value = neighbor_value
            path.append((current_x, current_value))
        # NO → Stay where we are. (This is why HC gets stuck!)
        #       It NEVER accepts a worse move.

    # === DONE ===
    return current_x, current_value, path


# ----- STEP 4: Random-Restart Hill Climbing -----
# Idea: "If at first you don't succeed, try, try again."
# Run HC multiple times from different starting points.
# Keep the BEST result across all runs.

def random_restart_hill_climbing(num_restarts=10, **kwargs):
    """
    Run hill_climbing() multiple times, keep the best result.
    
    WHY THIS WORKS:
        Each restart drops us at a DIFFERENT random spot.
        Even if one run gets stuck on a small hill,
        another run might start near the tallest peak.
    
    MATH INSIGHT:
        If each run has probability p of finding the global max,
        then after k restarts, failure probability = (1-p)^k
        → approaches 0 as k grows!
    """
    best_x     = None
    best_value = float('-inf')   # Start with worst possible
    best_path  = []

    for restart in range(num_restarts):
        x, value, path = hill_climbing(**kwargs)

        # Is this run's result better than our all-time best?
        if value > best_value:
            best_x, best_value, best_path = x, value, path

    return best_x, best_value, best_path


# ----- RUN IT -----
if __name__ == "__main__":
    # Single run
    x, val, path = hill_climbing()
    print(f"Single HC  → x={x:.4f}, f(x)={val:.4f}, steps={len(path)}")

    # 20 restarts
    x2, val2, path2 = random_restart_hill_climbing(num_restarts=20)
    print(f"Restart HC → x={x2:.4f}, f(x)={val2:.4f} (best of 20 runs)")

💡 Key Takeaway
Hill climbing is fast, simple, and uses O(1) memory. But it's like a near-sighted hiker — it only sees what's immediately around it. For problems with many local optima, you need something that occasionally accepts worse moves. Enter: Simulated Annealing. 👇

Simulated Annealing — The Chill Optimizer

🎭 The Analogy
You're searching for the best restaurant in a huge city. Early in the night (high temperature), you're adventurous — you'll try random shady alleys and weird food trucks. As the night goes on (temperature drops), you get pickier. By midnight, you're only going to places rated 4.9+. This "cool down from chaos to order" is simulated annealing — inspired by how blacksmiths cool metal slowly to make it stronger!
🔑 The Key Insight vs Hill Climbing
Unlike hill climbing, SA sometimes accepts worse moves! This lets it escape local optima. The probability of accepting a bad move decreases over time as the "temperature" cools.

The Acceptance Probability

When a neighbor is worse by an amount ΔE:

P(accept worse move) = e ΔE / T

T high → P ≈ 1 (accept almost anything — exploring!). T low → P ≈ 0 (only accept improvements — exploiting!).

Pseudocode

function SIMULATED-ANNEALING(problem, schedule):
  current ← problem.INITIAL-STATE
  for t = 1 to ∞:
    T ← schedule(t)  // temperature at time t
    if T = 0: return current
    next ← random neighbor of current
    ΔE ← VALUE(next) − VALUE(current)
    if ΔE > 0: current ← next  // better → always accept
    else: current ← next with probability eΔE/T  // worse → maybe accept

Cooling Schedules

ScheduleFormulaBehavior
LinearT = T₀ − α·tSteady decrease. Simple but crude.
GeometricT = T₀ · αt (α ≈ 0.95–0.99)Most popular. Smooth exponential decay.
LogarithmicT = T₀ / ln(t + 1)Theoretically guarantees global optimum (but very slow).

🐍 Python Code — Fully Explained

Pythonsimulated_annealing.py
# =====================================================
#  SIMULATED ANNEALING — Hill Climbing that can go downhill!
# =====================================================
#  WHAT IT DOES:
#    Like hill climbing, BUT it sometimes accepts WORSE moves.
#    This allows it to ESCAPE local optima.
#
#  THE BIG IDEA:
#    Early on  → high temperature → wild, accepts bad moves
#    Later    → low temperature  → calm, only accepts good moves
#
#  REAL-WORLD ORIGIN:
#    Inspired by metallurgy — heating metal then cooling slowly
#    lets atoms settle into the strongest crystal structure.
# =====================================================

import random, math


def objective(x):
    """Same wavy landscape with multiple peaks."""
    return math.sin(x) * math.cos(x / 2) * math.sin(x / 3)


def simulated_annealing(
    lower       = 0,       # Left boundary of search space
    upper       = 20,      # Right boundary of search space
    T_initial   = 1.5,     # Starting temperature (HIGH = more exploration)
    T_min       = 0.001,   # Stop when temperature falls below this
    cooling     = 0.995,   # Multiply T by this each step (0.995 = slow cool)
    step_size   = 0.5,     # How far neighbors can be (bigger = bolder jumps)
    max_iter    = 10000    # Safety stop
):
    """
    Simulated Annealing to MAXIMIZE the objective function.
    
    ALGORITHM IN PLAIN ENGLISH:
        1. Start at a random spot. Set temperature HIGH.
        2. Pick a random neighbor.
        3. If neighbor is BETTER → move there (always).
        4. If neighbor is WORSE  → MAYBE move there.
           The probability depends on:
             - HOW much worse (small difference → more likely)
             - Current temperature (hot → more likely)
           Formula: probability = e^(delta / temperature)
        5. Cool down the temperature slightly.
        6. Repeat until temperature is near zero.
    
    RETURNS:
        best_x, best_value  = the BEST solution ever seen
        history             = list of (x, value, temperature) for plotting
    """

    # === SETUP ===
    current_x     = random.uniform(lower, upper)
    current_value = objective(current_x)

    # IMPORTANT: Track the BEST-EVER solution separately!
    # Because SA can move AWAY from good solutions,
    # we need to remember the best we've ever found.
    best_x     = current_x
    best_value = current_value

    temperature = T_initial
    history     = [(current_x, current_value, temperature)]

    # === MAIN LOOP ===
    for step in range(max_iter):

        # Are we cold enough to stop?
        if temperature < T_min:
            break

        # Pick a random neighbor
        neighbor_x = current_x + random.uniform(-step_size, step_size)
        neighbor_x = max(lower, min(upper, neighbor_x))  # Stay in bounds
        neighbor_value = objective(neighbor_x)

        # How much better/worse is the neighbor?
        delta = neighbor_value - current_value
        # delta > 0 means neighbor is BETTER
        # delta < 0 means neighbor is WORSE

        # === THE CRITICAL DECISION ===
        if delta > 0:
            # BETTER → Always accept. No brainer!
            current_x     = neighbor_x
            current_value = neighbor_value

        else:
            # WORSE → Maybe accept, depending on temperature.
            #
            # acceptance_probability = e^(delta / temperature)
            #
            # Example with delta = -0.3:
            #   T = 1.0  → e^(-0.3/1.0) = 0.74  → 74% chance accept
            #   T = 0.1  → e^(-0.3/0.1) = 0.05  → 5% chance accept
            #   T = 0.01 → e^(-0.3/0.01) ≈ 0    → ~0% chance accept
            #
            # So: hot = adventurous, cold = picky

            acceptance_prob = math.exp(delta / temperature)

            if random.random() < acceptance_prob:
                # Accepted a WORSE move! (This is the magic of SA)
                current_x     = neighbor_x
                current_value = neighbor_value

        # Did we just find a new all-time best?
        if current_value > best_value:
            best_x     = current_x
            best_value = current_value

        # Cool down (this is the "annealing" part)
        temperature = temperature * cooling

        history.append((current_x, current_value, temperature))

    # === DONE ===
    return best_x, best_value, history


# ----- RUN IT -----
if __name__ == "__main__":
    x, val, hist = simulated_annealing()
    print(f"SA Result → x={x:.4f}, f(x)={val:.4f}")
    print(f"Total iterations: {len(hist)}")

Local Beam Search — The Search Party

🎭 The Analogy
Hill climbing sends one hiker. Random restart sends hikers one at a time. Local Beam Search says: "Send k hikers at once and give them walkie-talkies!" 📻 If one person finds a promising area, the whole team shifts toward it.
⚠️ Common Misconception Alert!
Local Beam Search is NOT running k independent hill climbers! The key difference: at each step, ALL successors of ALL k states are generated, and the best k overall are kept — regardless of which parent they came from. States share information.

Pseudocode

function LOCAL-BEAM-SEARCH(problem, k):
  Generate k random states
  loop:
    if any state is a goal: return it
    Generate ALL successors of all k states
    Pool all successors together
    Select the k best from the pool
    Replace current states with these k best

🐍 Python Code — Fully Explained

Pythonlocal_beam_search.py
# =====================================================
#  LOCAL BEAM SEARCH — k searchers working as a TEAM
# =====================================================
#  WHAT IT DOES:
#    Keep k states at all times. Each step, generate ALL
#    successors of ALL k states, then keep only the top k.
#
#  KEY DIFFERENCE FROM PARALLEL HC:
#    In parallel HC, each climber is independent.
#    In beam search, they SHARE — if one state's area
#    has great successors, the whole beam shifts there.
#
#  WEAKNESS:
#    All k states can CONVERGE to the same region,
#    losing diversity. (Fix: use stochastic beam search.)
# =====================================================

import random, math

def objective(x):
    return math.sin(x) * math.cos(x/2) * math.sin(x/3)


def make_neighbors(x, count=4, step=0.3, lo=0, hi=20):
    """Generate 'count' random neighbors of x within bounds."""
    neighbors = []
    for _ in range(count):
        nx = x + random.uniform(-step, step)
        nx = max(lo, min(hi, nx))           # Clamp to bounds
        neighbors.append(nx)
    return neighbors


def local_beam_search(
    k               = 5,     # Number of states to keep (beam width)
    neighbors_each  = 4,     # How many neighbors each state generates
    max_iterations  = 200,   # How many rounds
    lower           = 0,
    upper           = 20
):
    """
    Local Beam Search with k parallel states.
    
    ALGORITHM IN PLAIN ENGLISH:
        1. Start with k random positions (the "beam")
        2. Each position generates a few neighbors
        3. Collect ALL neighbors from ALL positions into one big list
        4. Sort by quality and keep only the TOP k
        5. These top k become the new beam
        6. Repeat
    
    EXAMPLE with k=3:
        Round 1: States at x=[2.0, 8.5, 15.1]
                 Each generates 4 neighbors → 12 new candidates
                 Plus 3 originals → 15 total candidates
                 Keep top 3: maybe [8.4, 8.6, 8.5] ← all near x=8!
        → This is the CONVERGENCE problem.
    """

    # STEP 1: Create k random starting positions
    beam = [random.uniform(lower, upper) for _ in range(k)]
    history = [list(beam)]  # Save each generation for visualization

    for iteration in range(max_iterations):

        # STEP 2: Generate ALL neighbors of ALL states in the beam
        all_candidates = list(beam)  # Start with current states (keep good ones)

        for state in beam:
            new_neighbors = make_neighbors(state, count=neighbors_each)
            all_candidates.extend(new_neighbors)
        # Now all_candidates has: k originals + k*neighbors_each new ones

        # STEP 3: Sort ALL candidates by quality (best first)
        all_candidates.sort(key=objective, reverse=True)

        # STEP 4: Keep only the top k → this is the new beam
        beam = all_candidates[:k]
        history.append(list(beam))

    # Return the single best state
    best = max(beam, key=objective)
    return best, objective(best), history


# =====================================================
#  STOCHASTIC BEAM SEARCH — fixes the diversity problem
# =====================================================
#  Instead of always picking the top k (deterministic),
#  we pick k states PROBABILISTICALLY — better states
#  are MORE LIKELY to be chosen, but weaker ones still
#  have a chance. Like natural selection!

def stochastic_beam_search(k=5, neighbors_each=4,
                           max_iterations=200, lower=0, upper=20,
                           temperature=0.5):
    """
    Like local beam search, but select survivors using
    WEIGHTED RANDOM selection instead of strict top-k.
    
    Weights come from softmax: weight_i = e^(value_i / temperature)
    Higher value → higher weight → more likely to survive.
    But even low-value states have a small chance → DIVERSITY!
    """
    beam = [random.uniform(lower, upper) for _ in range(k)]

    for _ in range(max_iterations):
        all_candidates = list(beam)
        for s in beam:
            all_candidates.extend(make_neighbors(s, count=neighbors_each))

        # Compute softmax weights
        values  = [objective(c) for c in all_candidates]
        max_val = max(values)
        weights = [math.exp((v - max_val) / temperature) for v in values]

        # Weighted random selection (higher fitness → more likely)
        beam = random.choices(all_candidates, weights=weights, k=k)

    best = max(beam, key=objective)
    return best, objective(best)


# ----- RUN IT -----
if __name__ == "__main__":
    x, val, hist = local_beam_search(k=6)
    print(f"Beam (k=6)       → x={x:.4f}, f(x)={val:.4f}")

    x2, val2 = stochastic_beam_search(k=6)
    print(f"Stochastic Beam  → x={x2:.4f}, f(x)={val2:.4f}")

Tabu Search — The Elephant That Never Forgets

🎭 The Analogy
You're exploring a maze. Normal hill climbing might walk you in circles — keep returning to the same "good-looking" dead end. Tabu Search keeps a "Do Not Visit" list of recent places. Even if a visited room looks attractive, you're forbidden from going back (for a while). This forces you to explore new territory!

Key Components

ComponentWhat It DoesExample
Tabu ListShort-term memory of recently visited states/movesLast 7 positions visited
Tabu TenureHow long a move stays forbiddenBanned for 10 iterations
Aspiration CriteriaOverride: allow a tabu move IF it's the new best ever"This tabu move is too good to ignore!"
⚠️ Critical: Tabu Search DOES Accept Worse Moves!
Tabu search picks the best non-tabu neighbor even if it's worse than the current state. This is essential — it prevents cycling and forces exploration. "You can't go back, so you MUST go forward, even downhill."

🐍 Python Code — Fully Explained

Pythontabu_search.py
# =====================================================
#  TABU SEARCH — Uses MEMORY to avoid revisiting states
# =====================================================
#  WHAT IT DOES:
#    Like hill climbing, but keeps a "banned list" of
#    recently visited states. Forces the search forward
#    into NEW territory, preventing cycling.
#
#  UNIQUE FEATURES:
#    1. Tabu List    — "no-go zones" that expire after a while
#    2. Aspiration   — override the ban if a move is AMAZING
#    3. Forced moves — MUST move, even downhill, if all good
#                      moves are tabu
#
#  BEST FOR: Combinatorial problems like TSP, scheduling
# =====================================================

import random, math
from collections import deque   # A list that auto-removes old items


def objective(x):
    return math.sin(x) * math.cos(x/2) * math.sin(x/3)


def tabu_search(
    lower       = 0,
    upper       = 20,
    tabu_tenure = 10,      # How many recent states to remember (ban duration)
    num_neighbors = 20,    # Candidates to evaluate each step
    max_iter    = 500,     # Maximum steps
    step_size   = 0.5,     # Neighbor generation range
    precision   = 1        # Rounding for tabu comparison (1 decimal place)
):
    """
    Tabu Search to MAXIMIZE the objective function.
    
    ALGORITHM IN PLAIN ENGLISH:
        1. Start at a random spot. Create an empty "banned" list.
        2. Generate many neighbors.
        3. Remove any neighbor that's in the banned list
           UNLESS it would be the new best-ever (aspiration).
        4. Move to the BEST non-banned neighbor.
           (Even if it's WORSE than current! You MUST move.)
        5. Add the new position to the banned list.
        6. If the banned list is too long, un-ban the oldest entry.
        7. Repeat.
    
    WHY ROUNDING?
        We round positions to 1 decimal place for tabu comparison.
        Without this, 5.001 and 5.002 would be "different" states,
        making the tabu list useless. Rounding creates "zones".
    """

    # === SETUP ===
    current_x     = random.uniform(lower, upper)
    current_value = objective(current_x)
    best_x        = current_x
    best_value    = current_value
    history       = [(current_x, current_value)]

    # The tabu list: a deque with max length = tenure.
    # When it's full and we add a new item,
    # the OLDEST item is automatically removed. Perfect!
    tabu_list = deque(maxlen=tabu_tenure)
    tabu_list.append(round(current_x, precision))

    # === MAIN LOOP ===
    for step in range(max_iter):

        # STEP 1: Generate candidate neighbors
        allowed_candidates = []  # Non-tabu options

        for _ in range(num_neighbors):
            nx = current_x + random.uniform(-step_size, step_size)
            nx = max(lower, min(upper, nx))
            nx_rounded = round(nx, precision)
            nval = objective(nx)

            # Is this position in the banned list?
            is_banned = (nx_rounded in tabu_list)

            # ASPIRATION CRITERION:
            # Even if it's banned, allow it if it's the NEW BEST EVER.
            # "The rule is 'no revisits', but this is TOO GOOD to pass up!"
            if is_banned and nval > best_value:
                is_banned = False   # Override! Aspiration activated!

            if not is_banned:
                allowed_candidates.append((nx, nval))

        # Edge case: all neighbors are tabu (very rare)
        if not allowed_candidates:
            nx = current_x + random.uniform(-step_size*2, step_size*2)
            nx = max(lower, min(upper, nx))
            allowed_candidates = [(nx, objective(nx))]

        # STEP 2: Pick the BEST allowed neighbor
        # NOTE: We take the best even if it's WORSE than current!
        # This is what prevents cycling — we MUST move forward.
        allowed_candidates.sort(key=lambda c: c[1], reverse=True)
        current_x, current_value = allowed_candidates[0]

        # STEP 3: Ban this new position
        tabu_list.append(round(current_x, precision))
        # (Oldest ban auto-expires because deque has maxlen)

        # STEP 4: Update global best
        if current_value > best_value:
            best_x, best_value = current_x, current_value

        history.append((current_x, current_value))

    return best_x, best_value, history


# ----- RUN IT -----
if __name__ == "__main__":
    x, val, hist = tabu_search()
    print(f"Tabu Search → x={x:.4f}, f(x)={val:.4f}")
    print(f"Iterations: {len(hist)}")

🍕 The Pizza Summary
Hill Climbing: Walk to the nearest pizza place. It's okay. You stop.
Simulated Annealing: Early on, randomly try places across town. Later, settle near the best one found.
Local Beam Search: Send 5 friends to different pizza places. Every round, everyone goes toward the best reports.
Tabu Search: Try pizza places but keep a list of the last 10 visited. Can't revisit! Forces you to discover new joints.

Bonus: Exam Day Cheat Sheet

🧠 "Which algorithm should I use?"
Smooth landscape, need speed? → Hill Climbing (+ random restarts)
Complex landscape, many local optima? → Simulated Annealing
Can afford parallel evaluations? → Local Beam Search
Combinatorial, cycling is an issue? → Tabu Search
No idea? → "Simulated Annealing" is usually a safe answer. 😉
